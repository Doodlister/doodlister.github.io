发表于 2020-01-20

基础概念

散列表

散列表)（Hash table，也叫哈希表），是根据键（Key）而直接访问在内存存储位置的数据结构。也就是说，它通过计算一个关于键值的函数，将所需查询的数据映射到表中一个位置来访问记录，这加快了查找速度。这个映射函数称做散列函数，存放记录的数组称做散列表。

负载因子

负载因子是：散列表的元素个数\散列表的长度。由于表长是定值，所以负载因子越大，代表填入的元素越多，那么发生冲突的概率也就越大。

拉链法

当发生冲突的时候，HashMap是采用拉链法来解决冲突的。

主要执行流程如下：

  1. 如果table为EMPTY_TABLE，那么先inflateTable
  2. 如果插入的key为null，那么对null值特殊处理（可见HashMap是允许插入null值的）
  3. 否则执行正常插入逻辑
    1. 先算出key的hash
    2. 找到hash对应table中的index
    3. 然后在对应index中的链表进行遍历
    4. 如果插入的key已经存在，则把其value替换为新值，并返回旧值
    5. 否则直接插入新的key

transfer方法主要是把旧数组的内容转移到新数组

主要流程如下：

  1. 遍历原来的老数组
  2. 针对老数组每个index对应的链表进行便利
  3. 如果需要reshah则重新计算hash
  4. 根据hash找到index
  5. 然后头插进链表

可以看到transfer是非常耗时的，每一次transfer需要遍历所有的Entry一次，时间复杂度 上升到了O(n)..

这个方法的主要作用就是对存入的key的hash进行二次hash，这样做的作用就是防止对象分布的不均匀，减少冲突。

* 因为table的长度都是2^n的形式，如果不进行rehash的话，如果两个对象Hash的低位相同，特别容易造成冲突如：

0101 0000 0000 1111 = 20495
0111 0000 0000 1111 = 28687

假设现在表长为16，现在直接算出其Table Index 分别为

  • 20495 %16 = 15

  • 28687 % 16 = 15

所以hash方法可以尽量让每一位都参与运算，尽量让相近的数字最后得到的hash分散开来。

可以看到新节点是采用头插法进行插入的。

插入方法的总结

  1. HashMap 初始化负载因子为0.75
  2. HashMap 初始化BucketTable长度为16
  3. BucketTable在第一次调用put方法的时候初始化
  4. 可以使用null作为key，且null的BucketTableIndex 为0
  5. 调用put方法的时候如果size>threadHold需要进行扩容(threadHold = newCapacity * loadFactory)
  6. 扩容需要进行rehash和BucketTable的复制，很耗费时间
